The Problem

Data URIs are a powerful way to embed entire resources (images, HTML, etc) in the link itself. For example, the link

data:text/html,<html><body><p>Hello world!</p></body></html>

displays an HTML page. Note that the HTML does not come from some external server, but from the link itself. Thus, you don't need a host to serve up HTML if you do it like this.

However, such links are long and clunky. If one could use a URL shortener, things would be great!

Unfortunately, URL shorteners do not accept Data URIs for security reasons.

The Solution

URI Redirect redirects requests from this webpage to any other page specified as a GET parameter.

This tool works around the limitation of URL shorteners not accepting data URIs. For example, this will shorten properly, since this looks like a normal webpage to the shortener.

https://sjhuang26.github.io/uri-redirect/r.html?uri=[[DATA URI]]

However, the shortened link will in turn redirect to the data URI specified.

Note: Data URIs (and thus the HTML) feature are not well supported in Internet Explorer.

Syntax

Redirect to URI

https://sjhuang26.github.io/uri-redirect/r.html?uri=[[URI]]

Redirect to HTML

This is just for convenience. It is just a URI redirect for data:text/html,[[HTML]].

https://sjhuang26.github.io/uri-redirect/r.html?html=[[HTML]]
